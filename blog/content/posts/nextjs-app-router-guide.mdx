## App Router vs Pages Router

Next.js 14 的核心变革：**React Server Components（RSC）成为默认**。

| 特性 | Pages Router | App Router |
|------|-------------|------------|
| 路由方式 | 文件即路由 | 文件夹 + page.tsx |
| 渲染模式 | CSR/SSR/SSG | **RSC 默认** |
| 布局嵌套 | `_app.tsx` | 多层 layout.tsx |
| 数据获取 | `getServerSideProps` | 组件内 `async/await` |
| 加载状态 | 手动管理 | `loading.tsx` 自动 |
| 错误边界 | `_error.tsx` | `error.tsx` 细粒度 |

## 文件约定速查

```
app/
├── layout.tsx        # 根布局（共享 UI）
├── page.tsx          # 首页
├── loading.tsx       # 加载骨架屏
├── error.tsx         # 错误边界
├── not-found.tsx     # 404 页面
├── posts/
│   └── [slug]/
│       ├── page.tsx  # /posts/hello-world
│       └── loading.tsx
└── api/
    └── route.ts      # API 路由
```

## Server Component 示例

```tsx
// app/posts/page.tsx —— 这是一个 Server Component
import { getAllPosts } from "@/lib/posts";

export default async function PostsPage() {
  // 直接在组件内 async/await，不需要 getServerSideProps
  const posts = getAllPosts(); // 读文件系统、调数据库都可以

  return (
    <ul>
      {posts.map((p) => (
        <li key={p.slug}>{p.frontmatter.title}</li>
      ))}
    </ul>
  );
}
```

## 禁用 SSR（需要交互的组件）

```tsx
"use client"; // 文件顶部加这一行，变成 Client Component

import { useState } from "react";

export function ThemeToggle() {
  const [dark, setDark] = useState(false);
  return <button onClick={() => setDark(!dark)}>切换</button>;
}
```

## Metadata API

```tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: "我的博客",
  description: "AI 前沿资讯与开发实践",
  openGraph: {
    title: "我的博客",
    description: "AI 前沿资讯与开发实践",
    type: "website",
  },
};
```

## 静态生成与 ISR

```tsx
// 静态生成所有文章页
export async function generateStaticParams() {
  const posts = getAllPosts();
  return posts.map((p) => ({ slug: p.slug }));
}

// 每 60 秒重新验证（ISR）
export const revalidate = 60;
```

> App Router 的核心理念：**能用 Server Component 就不要用 Client Component**。
